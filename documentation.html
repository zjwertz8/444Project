Project Documentation

Inspiration

Our team approached this topic with two goals in mind - create something that is genuinely useful to solve a problem, and have that problem be local. We were worried that if we chose a problem first, we would end up looking for a corresponding data set that would be predisposed to supporting our hypothesis. Consequently, we decided we wanted to try and find a dataset large enough to be an accurate representation of the population first. Then, we would use what weâ€™ve learned about information visualization to see if we could bring to light any trends among that data that were not already obvious.

We traversed campus, speaking to multiple different departments and programs to explain our project. Ultimately, we received a dataset of parking tickets from the Parking and Transportation Services department.

The Data

Due to privacy concerns, Parking and Transportation Services had to redact some of the data. We received 179284 tickets from 2014 to October 5th, 2017. Many of the tickets were voided and not considered. The full data set is avaiable below. Warning - the raw data sets are very large.

Data Sets

FullDataRaw.xlsx - Raw data saved as an excel sheet

FullDataRaw.csv - Raw data saved as a CSV file

In our representation of the data, we used the location by parking lot of each ticket as well as its date to create our visualizations.

What it does

Our application allows the user to view a heatmap representing the density of parking tickets across campus. By default, the map shows all of the available ticket data. The user can narrow or expand the range of data from the dropdown meu. There is an option to toggle the parking zones on and off as well as toggle the JAC's data (as the JAC skews the representation of the rest of the data).

The line graph shows the quantity of tickets across the ten most ticket-dense lots over the full dataset. The user must select which dots to view using the legend at the bottom.

The pie chart shows the comparison of the quantity of tickets across the different days of the week over the full data set or over multiple years.

Significant Milestones

Milestone 1 - Format the raw data.

Successful separation of excel data, able to retrieve lot name, 2014-2017 for year, 1-12 for month, 1-31 for day. Found in the file data.csv

Milestone 2 - Configured d3.js to interact with Google Maps and our data.

When using lot name, year, month, and day as the keys in thd3 rollup function, it creates an array of objects with keys and values. These values just so happen to be other arrays of objects with keys and values. This nest has four layers.

So with a little patience, and some intricate for loops, we are able to track each ticket to a parking lot regardless of what the date is.

The page now opens with google maps centered on campus. D3 functionality now works over the map.

Milestone 3 - Make polygons fill color with the data.

We drew polygons over each of the individual parking lots. The value for the number of tickets per parking lot is listed by individual lots over very specific spaces; there are over 200 lots and we needed the parking department to draw out the boundaries of each one.

We used a tool called Google Maps and KML shapes generator to draw the polygons. This tool also exported the raw latitude and longitude values for each of the points on our polygons. We took that data and put it into one of our reformatting tools (excel sheet and java program) to quickly convert it into the format needed in javascript to plot the polygon.

Milestone 4 - Adding selectors to change data.

Custom FillColor function returns correct value based on parameters. Have drop down list for year, program correctly stores drop down selection

Milestone 5 - Adding color scale and perfect color gradient.

Implemented use of a color gradient represents a luminosity transition and color transition that is friendly to users with color deficiencies.

Milestone 6 - Toggle the outlier JAC to avoid misleading visualization.

Because the JAC lot has twice as many tickets as any other lot, we added a toggle button to view the trends across campus excluding the JAC.

Milestone 7 -Line Graph to visualize the trend.

Included a line graph that plots the quantity of parking tickets for the top 10 lots across all available years.

Milestone 8 - Pie chart for Day comparison.

Included a pie chart that compares the quantity of tickets per weekday over all available years.

Milestone 9 - Adding zones to the whole map, and selector.

While the tickets identify the location of the ticket by individual days, the parking administration is more familiar with the tickets in terms of parking zones. We began implementation on a toggle for parking zones.
